NotebookLM セットアップ指示

これは、受け取った notebooklm_std フォルダをセットアップするための作業指示です。このページを読んだ AI（Claude Code 等）は、フォルダを開いた状態で以下を順に実行してください。

手順

  1. フォルダ内の notebooklm_raw.ipynb のセットアップ手順に従って環境を準備する。必要環境：Python 3.10 以上 / pip install httpx playwright / Google Chrome。
  2. 1箇所だけ修正：nlm_client.py 内の文字列 Invoke-NotebookLM login を python nlm_client.py login に全置換する。これは Cookie 未検出時に出るエラーメッセージの表記（古い表記の取り残し）で、機能には影響しないコスメ修正。行番号でなく文字列で探すこと（版により行番号が変わる＝同じ1箇所を指す）。
  3. python nlm_client.py login を実行し、会社の Google アカウントでログインする（会社の NotebookLM 契約を使う・2FA は画面の指示どおり）。Cookie が ~/.notebooklm/profiles/default/storage_state.json に保存される。
  4. 捨てノートブックで動作確認する：notebook を新規作成 → add_source_text でテキストを取り込み → ask で内容を質問 → 回答を確認 → その notebook を削除（後始末）。
  5. 最後に必ず「実行レポート」を出力する（下のテンプレートに沿って）。これはユーザーがコピーして共有する用なので、1つのコードブロックにまとめて、そのままコピペできる形で出力すること。
重要な制約：NotebookLM の操作はこのフォルダの nlm_client.py だけを使うこと。他の NotebookLM の MCP・ツールは併用しない（Cookie・認証の混線／ポート競合を避ける）。必要なら --profile <名前> と --port <番号> で隔離する。

最後に出力する実行レポート（テンプレート）

セットアップが完了（または途中で詰まっても）したら、以下を埋めて1つのコードブロックで出力してください。ユーザーがこれをコピーして共有し、フィードバックを受け取ります。事実のみ・推測は「不明」と書く。
